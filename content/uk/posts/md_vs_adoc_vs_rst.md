## Філософія

`Markdown` має бути настільки легким для читання та запису, наскільки це можливо.

Однак читабельність наголошується понад усе. Документ у форматі `Markdown` має бути доступним для публікації як є, як звичайний текст, без вигляду, ніби його розмічено тегами чи інструкціями щодо форматування. Хоча на синтаксис `Markdown` вплинули кілька існуючих фільтрів перетворення тексту в `HTML`, окрема `Setext`,  `atx`, `Textile`, `reStructuredText`, `Grutatext` і `EtText`, найбільшим джерелом  натхнення для синтаксису `Markdown` є формат електронної пошти простого тексту.

З цією метою синтаксис `Markdown` повністю складається із знаків пунктуації, які були ретельно підібрані, щоб виглядати як те, що вони означають. Наприклад, зірочки навколо слова насправді виглядають як \*виділення\*. Списки уцінки виглядають, ну, списки. Навіть блок-цитати виглядають як процитовані фрагменти тексту, якщо ви коли-небудь користувалися електронною поштою.



## Вбудований HTML

Синтаксис `Markdown` призначений для однієї мети: використовувати його як формат для написання для Інтернету.

`Markdown` не замінює `HTML` і навіть не наближається до нього. Його синтаксис дуже малий, він відповідає лише дуже невеликій підмножині тегів `HTML`. Ідея полягає не в тому, щоб створити синтаксис, який полегшить вставлення тегів `HTML`. На мій погляд, теги `HTML` вже легко вставити. Ідея `Markdown` полягає в тому, щоб полегшити читання, написання та редагування прози. `HTML` — це формат публікації; `Markdown` — це формат запису. Таким чином, синтаксис форматування `Markdown` вирішує лише проблеми, які можна передати звичайним текстом.

Для будь-якої розмітки, яка не охоплена синтаксисом `Markdown`, ви просто використовуєте сам `HTML`. Немає необхідності вводити перед ним або розмежовувати його, щоб вказати, що ви переходите з `Markdown` на `HTML`; ви просто використовуєте теги.


Єдині обмеження полягають у тому, що елементи `HTML` на рівні блоку, наприклад, `<div>`, `<table>`, `<pre>`, `<p>` тощо -- мають бути відокремлені від навколишнього вмісту порожніми рядками, а початкові та кінцеві теги блоку не мають мати відступи табуляції чи пробілів. `Markdown` достатньо розумний, щоб не додавати додаткові (небажані) теги `<p>` навколо тегів HTML-рівня блоку.

Наприклад, щоб додати таблицю HTML до статті `Markdown`:

```md
This is a regular paragraph.

<table>
    <tr>
        <td>Foo</td>
    </tr>
</table>

This is another regular paragraph.
```

Зауважте, що синтаксис форматування `Markdown` не обробляється в тегах `HTML` на рівні блоку. Наприклад, ви не можете використовувати \*виділення\* у стилі `Markdown` у блоці `HTML`.

Теги HTML рівня Span -- напр. `<span>`, `<cite>` або `<del>` — можна використовувати будь-де в абзаці `Markdown`, елементі списку або заголовку. Якщо ви хочете, ви навіть можете використовувати теги `HTML` замість форматування `Markdown`; напр. якщо ви віддаєте перевагу використанню тегів HTML `<a>` або `<img>` замість синтаксису посилань або зображень Markdown, продовжуйте.

На відміну від тегів `HTML` на рівні блоку, синтаксис `Markdown` обробляється в тегах рівня `span`.



## Автоматичне екранування для спеціальних символів

У HTML є два символи, які вимагають особливої обробки: `<` і `&`. Ліві кутові дужки використовуються для початку тегів; амперсанди використовуються для позначення сутностей `HTML`. Якщо ви хочете використовувати їх як буквальні символи, ви повинні екранувати їх як сутності, наприклад. `&lt;` і `&amp;`.

Амперсанди, зокрема, неприємні для авторів веб-сайтів. Якщо ви хочете написати про "AT&T", вам потрібно написати "AT&amp;T". Вам навіть потрібно екранувати амперсанд в URL-адресах. Таким чином, якщо ви хочете зробити посилання на:

http://images.google.com/images?num=30&q=larry+bird
вам потрібно закодувати URL як:

http://images.google.com/images?num=30&amp;q=larry+bird
у вашому атрибуті href тегу прив’язки. Зайве говорити, що це легко забути, і це, ймовірно, найпоширеніше джерело помилок перевірки HTML на добре розмічених веб-сайтах.

Markdown дозволяє вам використовувати ці символи природним чином, піклуючись про всі необхідні екранування за вас. Якщо ви використовуєте амперсанд як частину сутності HTML, він залишається незмінним; інакше його буде перекладено на &amp;.

Отже, якщо ви хочете включити у свою статтю символ авторського права, ви можете написати:


{{< tabs Markdown AsciiDoc reStructuredText >}}
  {{< tab >}}

  {{< /tab >}}
  {{< tab >}}

  {{< /tab >}}
  {{< tab >}}

  {{< /tab >}}
{{< /tabs >}}
