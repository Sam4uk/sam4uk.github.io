# 

Я давно збирався зробити шпаргалку для себе по користуванняю `make` не тільки, як зібрати чужий проект.
<!--https://makefiletutorial-com.translate.goog/?_x_tr_sl=auto&_x_tr_tl=en&_x_tr_hl=uk#makefile-cookbook-->
## Починаємо
### Для чого існують Makefile?

`Make`-файли використовуються, щоб допомогти вирішити, які частини великої програми потрібно перекомпілювати. У переважній більшості випадків файли `C` або `C++` компілюються. Інші мови зазвичай мають власні інструменти, які служать подібним цілям, як `Make`. `Make` також можна використовувати поза компіляцією, коли вам потрібна серія інструкцій для запуску залежно від того, які файли було змінено. У цьому підручнику буде зосереджено на прикладі використання компіляції `C/C++`.

Ось приклад графіка залежностей, який можна побудувати за допомогою Make. Якщо будь-які залежності файлу змінюються, файл буде перекомпільовано:

### Версії та типи Make

Існує багато реалізацій Make, але більша частина цього посібника працюватиме на будь-якій версії, яку ви використовуєте. Однак він спеціально написаний для GNU Make, яка є стандартною реалізацією в Linux і MacOS. Усі приклади працюють для версій 3 і 4 Make, які майже еквівалентні, за винятком деяких езотеричних відмінностей.

### Приклади

Щоб запустити ці приклади, вам знадобиться термінал і встановлений `make`. Для кожного прикладу помістіть вміст у файл із назвою `Makefile`, а в цьому каталозі виконайте команду make. Почнемо з найпростішого `Makefile`:

> Примітка. Файли Make-файлів мають мати відступи за допомогою символів TAB, а не пробілів, інакше make не вдасться.


```Makefile
hello:
	echo "Hello, World"
```

Ось результат виконання наведеного вище прикладу:

```zsh
$ make
echo "Hello, World"
Hello, World
```

### Синтакси Makefile

Makefile складається з набору правил . Загалом правило виглядає так:

```Makefile
targets: prerequisites
	command
	command
	command
```
- `targets`  є імена файлів розіделених пробілами. Як правило, для кожного правила існуєодна ціль.
- `command`  це серія кроків, які виконуються для досягення цілі. Вони мають починатися з символу табуляції, а не пробілів.
- `prerequisites` передумови (це також імна файлів) розділені пробілами. Ці файли мають існувати перед виконаням цільоваго файлу.
  
### суть Make

Почнемо з прикладу hello world:

```Makefile
hello:
	echo "Hello, World"
	echo "This line will print if the file hello does not exist."
```

- У нас є одна названа ціль `hello`
- Ця мета має дві _команди_
- Ця мета не має _передумов_

Тоді ми виконуємо `make hello`. Поки `hello`-файл не існує, команди виконуватимуться. Якщо `hello` існує, жодні команди не виконуватимуться.

Важливо розуміти, що я говорю `hello` як про ціль , так і про файл . Це тому, що вони безпосередньо пов’язані разом. Як правило, під час запуску цілі (також коли виконуються команди цілі), команди створюють файл із тим же ім’ям, що й ціль. У цьому випадку `hello` мета не створює `hello` файл.

Давайте створимо більш типовий `Makefile` - такий, який компілює один `C`-файл. Але перш ніж ми це зробимо, створіть файл під назвою, `blah.c` який має наступний вміст:

```cpp
// blah.c
int main() { return 0; }
```

Потім створіть `Makefile`:

```Makefile
blah:
	cc blah.c -o blah
```
Цього разу спробуйте просто виконати `make`. Оскільки ціль не вказана як аргумент команди `make`, виконується перша ціль. У цьому випадку є лише одна ціль (`blah`). Під час першого запуску `blah` буде створено. Другий раз побачиш make: `"blah" не вимагає оновлення`. Це тому, що `blah` файл уже існує. Але є проблема: якщо ми змінюємо `blah.c`, а потім запускаємо make, нічого не буде перекомпільовано.

Ми вирішуємо це, додаючи передумову:
```Makefile
blah: blah.c
	cc blah.c -o blah
```
Коли ми запускаємо `make` знову, відбувається наступні кроки:

- Вибрано першу ціль, оскільки перша ціль є ціллю за замовчуванням
- Для цього є передумова `blah.c`
- Make вирішує, чи слід запускати `blah` ціль. Він запускатиметься, лише якщо `blah` не існує або `blah.c` є новішим за `blah`

### Чистимо

`clean` часто використовується як ціль, яка видаляє вихідні дані інших цілей, але це не спеціальне слово в `make`. Ви можете запускати `make` і `make clean` на цьому створити та видалити `some_file`.

Зверніть увагу, що `clean` тут робиться дві нові речі:
- Це мета, яка не є першою (за замовчуванням) і не є обов’язковою умовою. Це означає, що він ніколи не запуститься, якщо ви явно не запустите `make clean`
- Воно не призначене для імені файлу. Якщо у вас є файл з іменем clean, ця ціль не запуститься, чого ми не хочемо. Дивіться [.PHONY](#phony) далі в цьому підручнику, як це виправити.

```Makefile
some_file: 
	touch some_file

clean:
	rm -f some_file
```

### Зміні

Змінні можуть бути лише рядками. Зазвичай ви можете використовувати `:=`, але `=` також працює. Див. [Змінні, частина 2](#змінні-ч2).

Ось приклад використання змінних:
```Makefile
files := file1 file2
some_file: $(files)
	echo "Look at this variable: " $(files)
	touch some_file

file1:
	touch file1
file2:
	touch file2

clean:
	rm -f file1 file2 some_file
```

Одинарні або подвійні лапки не має різниці. Це просто символи, які призначаються змінній. Однак лапки корисні для `shell/bash`, і вони потрібні вам у таких командах, як `printf`. У цьому прикладі дві команди поводяться однаково:
```Makefile
a := one two # a is set to the string "one two"
b := 'one two' # Not recommended. b is set to the string "'one two'"
all:
	printf '$a'
	printf $b
```

Посилання на змінні з використанням `${}` або `$()`

```Makefile
x := dude

all:
	echo $(x)
	echo ${x}

	# Bad practice, but works
	echo $x
```

## Цілі
### Всі
Створюєте кілька цілей і хочете, щоб усі вони були виконані? Зробіть `all` ціль. Оскільки це перше правило в списку, воно виконуватиметься за замовчуванням, якщо `make` викликатиметься без вказівки цілі.

```Makefile
all: one two three

one:
	touch one
two:
	touch two
three:
	touch three

clean:
	rm -f one two three
```

### Кілька цілей

Якщо для правила є кілька цілей, команди виконуватимуться для кожної цілі. `$@` це автоматична змінна , яка містить цільове ім’я.

```Makefile
all: f1.o f2.o

f1.o f2.o:
	echo $@
# Тотожне:
# f1.o:
#	 echo f1.o
# f2.o:
#	 echo f2.o

```
